# TaeeunKil Notes — Design System

이 문서는 `TaeeunKil.github.io`의 시각적 기준이다. Claude의 따뜻한 편집 디자인에서 출발하지만, 특정 회사의 브랜드를 복제하지 않는다. 모든 페이지는 프레임워크 없이 HTML과 CSS만으로 구현할 수 있어야 한다.

## 1. Visual Thesis

오래 보관한 노트처럼 따뜻하고 조용하지만 낡아 보이지 않는 디지털 기록실.

- 화면보다 종이에 가까운 분위기를 만든다.
- 글이 주인공이며 인터페이스는 뒤로 물러난다.
- 개인적인 흔적과 불규칙성을 허용하되 읽기 규칙은 일관되게 유지한다.
- 장식보다 여백, 글자 크기, 정렬과 선으로 위계를 만든다.
- 빠르게 소비하는 피드가 아니라 천천히 읽고 다시 찾는 아카이브를 지향한다.

## 2. Core Principles

1. **Content first** — 본문을 가리는 장식, 고정 UI, 팝업을 만들지 않는다.
2. **One accent** — 강조색은 테라코타 한 가지를 기본으로 한다.
3. **Cardless by default** — 글 목록은 카드 그리드가 아니라 행, 여백, 구분선으로 구성한다.
4. **No dependency required** — 외부 UI 라이브러리나 웹 폰트가 없어도 의도한 인상이 유지되어야 한다.
5. **Progressive enhancement** — JavaScript가 꺼져도 모든 글과 링크를 읽고 이동할 수 있어야 한다.
6. **Each post may differ** — 개별 글은 주제에 맞는 색, 이미지, 조판을 가질 수 있지만 본문 폭과 접근성 규칙은 지킨다.

## 3. Color System

```css
:root {
  --paper: #f5f1e8;
  --paper-raised: #fbf8f1;
  --ink: #1c1b18;
  --ink-soft: #49463f;
  --muted: #756f65;
  --line: #d8d1c5;
  --accent: #b8583c;
  --accent-dark: #873d2b;
  --code-bg: #eae4d9;
  --focus: #1769aa;
}
```

- 기본 배경은 `--paper`, 본문은 `--ink`를 사용한다.
- 순백과 순흑은 이미지나 특별한 대비가 필요한 경우에만 사용한다.
- `--accent`는 링크, 현재 위치, 작은 강조에만 사용한다. 넓은 배경에는 사용하지 않는다.
- 접근성 포커스는 브랜드 색과 분리된 `--focus`를 사용한다.
- 그라데이션은 사용하지 않는다.

## 4. Typography

별도 다운로드 없이 운영체제 글꼴로 읽기 좋은 조합을 만든다.

```css
:root {
  --font-display: Georgia, "Times New Roman", "Noto Serif KR", Batang, serif;
  --font-body: system-ui, -apple-system, "Segoe UI", "Noto Sans KR", sans-serif;
  --font-mono: ui-monospace, "Cascadia Code", Consolas, monospace;
}
```

| 역할 | 크기 | 굵기 | 줄 높이 | 서체 |
|---|---:|---:|---:|---|
| 사이트 이름 | `clamp(2.6rem, 8vw, 6.5rem)` | 400 | 0.95 | Display |
| 페이지 제목 | `clamp(2.2rem, 6vw, 4.5rem)` | 400 | 1.05 | Display |
| 글 제목 | `clamp(2rem, 5vw, 3.75rem)` | 400 | 1.12 | Display |
| 섹션 제목 | `clamp(1.4rem, 3vw, 2rem)` | 400 | 1.2 | Display |
| 본문 | `clamp(1rem, 1.2vw, 1.125rem)` | 400 | 1.8 | Body |
| 메타데이터 | `0.8rem` | 500 | 1.4 | Body |
| 코드 | `0.9em` | 400 | 1.65 | Mono |

- 제목에는 과도한 굵기를 사용하지 않는다. 크기와 여백으로 위계를 만든다.
- 한국어 본문은 한 줄에 약 35–45자를 목표로 한다.
- 영문 대문자 라벨에는 `0.08em` 정도의 자간을 허용한다.
- 본문 문단 간격은 최소 `1.25em`으로 둔다.

## 5. Layout

- 전체 페이지 최대 폭: `72rem`.
- 본문 읽기 폭: `42rem` 또는 `68ch` 이하.
- 데스크톱 바깥 여백: 최소 `2rem`; 모바일: 최소 `1.25rem`.
- 주요 섹션 간격: 데스크톱 `6rem–9rem`, 모바일 `3.5rem–5rem`.
- 첫 화면은 사이트 이름과 소개 문장이 주 시각 요소가 되는 타이포그래피 포스터처럼 구성한다.
- 좌측 정렬을 기본으로 하며, 전체 화면 중앙 정렬은 짧은 선언문에만 사용한다.
- 구분선은 `1px solid var(--line)`만 사용한다.
- 기본 UI에는 박스 그림자를 사용하지 않는다.

## 6. Homepage Structure

홈은 네 가지 역할만 가진다.

1. **Masthead** — `TaeeunKil`이라는 이름과 블로그를 설명하는 한 문장.
2. **Latest writing** — 날짜, 제목, 짧은 설명으로 된 최신 글 목록.
3. **Archive / topics** — 연도 또는 주제로 과거 글에 접근하는 단순한 링크.
4. **About** — 운영자 소개와 GitHub 등 최소한의 연락 경로.

글 목록 한 행은 다음 순서를 따른다.

```text
2026.07.22    글 제목
              한두 줄의 짧은 설명
```

- 썸네일은 필수가 아니다.
- 모든 글을 같은 높이의 카드에 가두지 않는다.
- 최신 글 하나만 크기를 키울 수 있지만 나머지 글을 압도하지 않게 한다.

## 7. Article Pages

- 제목 위에는 날짜와 주제만 표시한다.
- 제목 아래에는 필요할 때만 한 문장 요약을 둔다.
- 본문 시작 전 장식용 구분선이나 작성자 카드가 필요하지 않다.
- `h2`와 `h3` 위에는 충분한 여백을 두고, 네 단계 이상의 제목 계층은 피한다.
- 인용문은 큰 따옴표 장식 대신 왼쪽 선과 여백으로 구분한다.
- 각주는 본문보다 작게 쓰되 대비를 지나치게 낮추지 않는다.
- 이전 글과 다음 글 링크는 본문 끝의 단순한 두 열 또는 세로 목록으로 둔다.

## 8. Links and Interaction

- 본문 링크는 색상만으로 구분하지 않고 항상 밑줄을 표시한다.
- 탐색 링크는 hover 시 밑줄이 왼쪽에서 오른쪽으로 짧게 나타날 수 있다.
- 클릭 가능한 모든 요소는 키보드 포커스를 명확히 표시한다.
- 버튼이 꼭 필요하면 사각형에 가까운 `3px` 반경을 사용한다. pill 버튼은 사용하지 않는다.
- 터치 영역은 최소 `44px`를 확보한다.

## 9. Motion

모션은 읽기 흐름을 설명할 때만 사용한다.

- 첫 진입: 제목과 소개가 `300–450ms` 동안 약하게 나타난다.
- 글 목록: 화면에 들어올 때 짧은 순차 노출을 허용한다.
- 링크: 밑줄 위치 또는 색상만 `150–200ms` 동안 변화한다.
- 긴 글: 상단에 얇은 읽기 진행선을 선택적으로 사용할 수 있다.
- 패럴랙스, 자동 재생, 무한 반복 장식 애니메이션은 사용하지 않는다.
- `prefers-reduced-motion: reduce`에서 모든 비필수 모션을 제거한다.

## 10. Images and Media

- 이미지는 글의 의미를 전달할 때만 사용하며 빈 공간을 채우기 위해 넣지 않는다.
- 사진은 본문 폭, 넓은 폭, 전체 폭 중 하나를 의도적으로 선택한다.
- 모든 의미 있는 이미지에는 대체 텍스트를 제공한다.
- 이미지 모서리를 습관적으로 둥글게 만들거나 그림자를 넣지 않는다.
- 캡션은 이미지 바로 아래에 `--muted` 색으로 배치한다.

## 11. Code and Technical Writing

- 인라인 코드는 `--code-bg` 배경과 `2px` 반경을 사용한다.
- 코드 블록은 가로 스크롤을 허용하고 줄 바꿈으로 코드를 훼손하지 않는다.
- 코드 블록 위에는 언어명 또는 파일명을 작은 모노스페이스 라벨로 표시할 수 있다.
- 구문 강조 색상은 4개 이하로 제한하고 본문 팔레트와 충돌하지 않게 한다.
- 복사 버튼은 JavaScript가 있는 경우에만 보조 기능으로 제공한다.

## 12. Responsive Behavior

- `48rem` 이하에서 탐색은 두 줄 또는 세로 목록으로 자연스럽게 흐른다. 햄버거 메뉴는 필요할 때만 사용한다.
- 글 목록은 날짜 열과 제목 열에서 단일 열로 바뀐다.
- 제목은 `clamp()`로 축소하며 모바일에서 강제 줄바꿈을 넣지 않는다.
- 표는 가로 스크롤 또는 의미를 보존하는 세로 구조로 변환한다.
- 모바일에서도 본문 줄 높이와 문단 간격을 줄이지 않는다.

## 13. Accessibility and Performance

- 본문과 배경의 명암 대비는 WCAG AA 이상이어야 한다.
- 의미 구조는 `header`, `nav`, `main`, `article`, `footer` 요소로 표현한다.
- 제목 단계는 건너뛰지 않는다.
- 현재 페이지 링크에는 `aria-current="page"`를 사용한다.
- 핵심 콘텐츠는 HTML에 직접 존재해야 하며 JavaScript fetch 성공 여부에만 의존하지 않는다.
- 이미지에는 크기를 명시해 레이아웃 이동을 막고 필요하면 `loading="lazy"`를 사용한다.
- 외부 요청과 전체 JavaScript 크기를 가능한 한 0에 가깝게 유지한다.

## 14. Do / Do Not

### Do

- 따뜻한 종이색과 차분한 잉크색을 유지한다.
- 여백으로 글 사이의 호흡을 만든다.
- 글마다 작은 개성을 허용한다.
- 시스템 폰트와 표준 HTML을 우선한다.
- 디자인 결정을 콘텐츠의 읽기 경험으로 설명한다.

### Do Not

- Claude의 로고, 이름, 전용 폰트 또는 일러스트 스타일을 복제하지 않는다.
- SaaS 랜딩 페이지 같은 카드 그리드와 CTA 묶음을 만들지 않는다.
- 장식용 그라데이션, 유리 효과, 큰 그림자를 사용하지 않는다.
- 태그와 배지를 과도하게 늘리지 않는다.
- 모든 글을 동일한 템플릿에 강제로 가두지 않는다.
- JavaScript가 없으면 비어 보이는 페이지를 만들지 않는다.

## 15. Agent Handoff

이 저장소를 수정하는 에이전트는 다음 순서를 따른다.

1. 기존 콘텐츠와 HTML의 의미 구조를 먼저 보존한다.
2. 이 문서의 색상, 조판, 여백 규칙을 CSS 변수로 구현한다.
3. 카드, 그림자, 색상을 추가하기 전에 타이포그래피와 정렬로 해결한다.
4. 새 컴포넌트는 키보드, 작은 화면, JavaScript 비활성 상태에서 확인한다.
5. 브랜드를 흉내 내기보다 `TaeeunKil`의 기록이 더 잘 읽히는지를 기준으로 판단한다.
