# TaeeunKil — Design System

이 문서는 `TaeeunKil.github.io`의 시각적 기준이다. 목표는 단순하다. **하나의 서체, 하나의 배경, 하나의 포인트 컬러로 글과 사람을 또렷하게 보여준다.**

## 1. Design Philosophy

- 심플함은 비어 있음이 아니라 우선순위가 명확한 상태다.
- 색보다 크기, 굵기, 정렬, 여백으로 위계를 만든다.
- 포인트 컬러는 사용자의 시선이 가야 할 한 곳에만 쓴다.
- 홈, 글, 소개는 긴 한 페이지의 섹션이 아니라 같은 공간을 공유하는 세 개의 탭이다.
- 장식보다 반응성과 읽기 경험을 우선한다.

## 2. Visual Thesis

밝은 종이 위에 검은 활자와 한 번의 주황색 표시만 남긴 현대적인 개인 기록실.

## 3. Color

```css
:root {
  --background: #f2f1ed;
  --text: #171716;
  --muted: #777570;
  --line: #d6d4ce;
  --accent: #e74b2b;
  --focus: #171716;
}
```

- `--accent`는 현재 탭, 링크 상태, 마침표 같은 작은 표식에만 사용한다.
- 회색은 중립적인 정보 계층에만 사용하며 별도의 보조색으로 취급하지 않는다.
- 반전 섹션, 그라데이션, 다색 태그를 사용하지 않는다.
- 그림자를 사용하지 않는다.

## 4. Typography

제목과 본문 모두 동일한 시스템 산세리프를 사용한다.

```css
--font: system-ui, -apple-system, "Segoe UI", "Apple SD Gothic Neo",
  "Noto Sans KR", "Malgun Gothic", sans-serif;
```

- 사이트 이름: `clamp(4.5rem, 16vw, 13rem)`, 700, 줄 높이 0.82.
- 페이지 제목: `clamp(2.8rem, 8vw, 6rem)`, 700, 줄 높이 0.95.
- 본문: `1rem–1.125rem`, 400, 줄 높이 1.75.
- 메타데이터: `0.72rem`, 600, 대문자와 넓은 자간.
- 글자 굵기는 400, 600, 700 세 단계만 사용한다.
- 제목과 본문을 다른 글꼴로 꾸미지 않는다.

## 5. App Shell

화면은 세 영역으로 구성한다.

1. 상단: 로고, `홈 / 글 / 소개` 탭, GitHub 링크.
2. 중앙: 선택된 탭의 콘텐츠 한 면.
3. 하단: 현재 탭과 제작 방식을 보여주는 작은 상태 표시줄.

- 데스크톱에서는 전체 높이를 `100svh`로 유지한다.
- 각 탭의 내용이 화면보다 길 때 해당 패널 내부만 스크롤한다.
- 탭 콘텐츠를 카드나 별도 컨테이너로 감싸지 않는다.

## 6. Tabs

- 탭은 일반 링크로 작성해 JavaScript가 없어도 해당 섹션으로 이동할 수 있어야 한다.
- JavaScript가 있으면 세 패널을 수평으로 이동시킨다.
- 현재 탭은 포인트 컬러의 작은 점과 `aria-selected`로 표시한다.
- 주소 해시(`#home`, `#writing`, `#about`)를 현재 상태와 동기화한다.
- 좌우 방향키로 탭 사이를 이동할 수 있어야 한다.

## 7. Home

- `TaeeunKil` 이름이 화면의 주 시각 요소다.
- 소개 문장은 한 문장만 둔다.
- 작은 포인트 점 하나 이외의 장식은 넣지 않는다.
- 홈에는 글 목록이나 소개 전문을 중복해 보여주지 않는다.

## 8. Writing

- 글은 카드가 아닌 가로 행 목록으로 표시한다.
- 각 행은 번호, 날짜, 제목, 화살표 순서로 구성한다.
- hover 시 제목과 화살표만 포인트 컬러로 변한다.
- 글이 없을 때는 가짜 글 대신 준비 중이라는 상태를 그대로 표시한다.

## 9. About

- 짧은 소개와 관심사, 외부 링크만 둔다.
- 프로필 카드, 통계, 기술 로고 모음을 사용하지 않는다.
- 문장은 넓고 크게 배치해 읽는 행위 자체가 화면 구성이 되게 한다.

## 10. Motion

- 첫 진입 시 이름이 `350ms` 동안 짧게 올라온다.
- 탭 전환은 `380–450ms`의 수평 이동으로 표현한다.
- 링크 상태 변화는 `150ms` 안에 끝낸다.
- 자동 재생, 반복 애니메이션, 패럴랙스를 사용하지 않는다.
- `prefers-reduced-motion`에서는 모든 이동을 즉시 전환한다.

## 11. Responsive Behavior

- 모바일에서는 로고를 축약하고 GitHub 링크를 숨길 수 있다.
- 탭 이름은 항상 남긴다.
- 사이트 이름은 여러 줄로 자연스럽게 나눌 수 있다.
- 글 목록은 날짜와 제목이 세로로 쌓이도록 바뀐다.
- 최소 터치 영역은 `44px`다.

## 12. Accessibility and Resilience

- 의미 구조에 `header`, `nav`, `main`, `section`, `footer`를 사용한다.
- 포인트 컬러만으로 현재 상태를 전달하지 않는다.
- 키보드 포커스를 명확히 표시한다.
- JavaScript가 없을 때 모든 패널을 세로로 표시한다.
- 핵심 콘텐츠는 JSON fetch 성공 여부에 의존하지 않는다.
- 외부 폰트, UI 라이브러리, 프레임워크는 사용하지 않는다.

## 13. Do / Do Not

### Do

- 한 화면에 하나의 메시지만 보여준다.
- 여백과 정렬로 요소를 구분한다.
- 포인트 컬러가 쓰인 이유를 설명할 수 있게 한다.
- 같은 서체를 크기와 굵기로 변주한다.

### Do Not

- 세리프와 산세리프를 섞지 않는다.
- 카드 그리드, pill 버튼, 큰 그림자를 사용하지 않는다.
- 탭 내용을 홈 아래에 반복해서 배치하지 않는다.
- 장식용 이미지나 아이콘을 추가하지 않는다.
- 두 번째 포인트 컬러를 만들지 않는다.
